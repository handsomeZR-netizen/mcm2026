## 0. 目标与交付

- **目标**：清晰、可复现、可解释，冲击高分。
- **交付**：不超过 25 页 PDF（含 Summary Sheet、目录、正文、1–2 页备忘录、参考文献、AI 使用报告）。

---

## 1. 题目关键信息（必须对齐）

- DWTS 美国版已完成 **34 季**；每周 **评委打分 + 观众投票** 决定淘汰。
- **投票是未知量**（商业机密）；需估计每周/每位选手的投票数或投票份额。
- 合并方法存在分段与不确定性：  
  - **S1–S2：排名法（Ranks）**  
  - **S3–S27：百分比法（Percentages）**  
  - **S28–S34：Bottom Two + Judges’ Save**（且可能恢复排名法）  
  → **需要做敏感性分析**（排名法 vs 百分比法）。
- 数据包含选手信息、赛季结果、每周评委评分；淘汰后周评分为 0。

---

## 2. 数据与对象口径

- **分析对象**：每个 *Season–Week* 中“仍在场”的搭档（当周有有效评委分）。
- **评委总分**：当周所有有效评委分相加（忽略 N/A）。
- **淘汰者**：以数据的 `results/placement` 标记为准；双淘汰为集合大小 2。
- **投票尺度**：投票总量未知 → 主要建模投票份额/相对强度，允许整体尺度自由。

---

## 3. 规则口径（默认主模型）

### 3.1 S1–S2：排名法

- `judge_rank`：按评委总分排序（高分 = rank 1）  
- `fan_rank`：按观众票排序（未知）  
- `combined_rank = judge_rank + fan_rank`  
**淘汰者 = combined_rank 最大者**；平局时 **观众票更少者淘汰**。

### 3.2 S3–S27：百分比法

- `judge_pct(i) = judge_total(i) / sum_judge_total`  
- `fan_pct(i) = fan_votes(i) / sum_fan_votes`  
**`combined = 0.5*judge_pct + 0.5*fan_pct`**  
**淘汰者 = combined 最小者**；平局时 **观众票更少者淘汰**。

### 3.3 S28–S34：Bottom Two + Judges’ Save

- 先计算 `combined`（主模型默认 **百分比法**；敏感性分析用 **排名法**）
- 取 `combined` 最低两对为 **Bottom Two**
- **真实淘汰者必须属于 Bottom Two**（但不一定是最低者）
- 若双淘汰：用 **Bottom K（K=3/4）** 做敏感性分析覆盖特殊周

---

## 4. 模型评价口径（可写进论文）

- **一致性（Consistency）**：
  - Week-level：真实淘汰者是否为预测最低者  
  - Bottom-level：真实淘汰者是否落在预测 Bottom 2  
  - Season-level：周一致率 / 赛季平均一致率  
- **确定性（Certainty）**：
  - 置信区间/后验区间宽度  
  - 预测分布熵或 Bootstrap 稳健性  
  - 选手与周之间的异质性比较  

---

## 5. 分析任务口径

- **对比两种合并方法**：在每个赛季同时应用排名法与百分比法  
- **争议选手分析**：Jerry Rice / Billy Ray Cyrus / Bristol Palin / Bobby Bones  
- **规则变化影响**：引入 Bottom Two + Judges’ Save 的差异  
- **特征影响分析**：年龄、行业、州/国家等对评委分与投票的影响差异  
- **提出更公平/更激励的规则**：需量化指标与解释支持

## 6. 写作与呈现标准

- 明确 **假设清单**、**符号表**、**公式与单位**  
- 所有外部数据 **完整引用来源**  
- 关键结论需有 **图表 + 指标** 支撑  
- 备忘录强调 **建议与可执行结论**

## 7. VibeCoding 工作流（协作标准）

1) **对齐问题与口径** → 2) **小步原型** → 3) **诊断与可视化**  
4) **敏感性/对照实验** → 5) **结论沉淀与写作**

- 每次口径变更记录在 `DECISIONS.md`  
- 输出物统一放入 `outputs/`，图表放入 `figures/`
