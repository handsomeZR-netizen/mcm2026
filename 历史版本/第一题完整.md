# 2026 MCM Problem C: 任务一完整建模方案

**核心方法论：基于近似贝叶斯计算 (ABC) 的观众投票反演与不确定性量化**

## 1. 核心叙事逻辑 (The Story)

我们将“重建观众投票”定义为一个**贝叶斯逆问题 (Inverse Problem)**。由于观众投票 $V$ 是黑箱，且淘汰机制 $f(J, V)$ 是非线性的（涉及排名、Bottom 2 裁决），无法写出解析解。因此，我们采用 **ABC-Rejection Sampling (近似贝叶斯计算-拒绝采样)** 算法：

1. **先验 (Prior):** 基于选手特征（非未来数据）构建“理论人气分布”。
2. **似然模拟 (Simulation):** 生成海量可能的投票分布，通过比赛规则进行“碰撞测试”。
3. **后验 (Posterior):** 保留那些能重现“真实淘汰结果”的样本，形成对真实票数的后验估计。

### 模块 B：无泄漏的先验构建 (Leakage-Free Prior Construction)

**目标：** 构建 $Prior \sim Dirichlet(\alpha)$，但绝不使用“当季存活周数”这种未来信息。

我们定义第 $t$ 周选手 $i$ 的**先验人气强度** $\alpha_i^{(t)}$ 为：

$$\alpha_i^{(t)} = \exp(\beta_0 + \beta_1 \cdot I_i + \beta_2 \cdot A_i + \beta_3 \cdot Trend_i^{(t)})$$

- **$I_i$ (Industry Score):** 职业加成。
  - *算法：* 使用 **“留一法” (Leave-One-Out)** 或 **“历史累计法”**。计算 S1 到 $S_{current-1}$ 赛季中，该职业选手的平均排名。**绝对不要用当前赛季的数据。**
- **$A_i$ (Age Score):** 年龄加成。同上，利用历史赛季数据回归得出。
- **$Trend_i^{(t)}$ (Momentum):** 动态表现（截止到 $t$ 周）。
  - 算法：$\frac{\text{本周评委分} - \text{前三周评委均分}}{\text{前三周评委均分}}$。捕捉“黑马”趋势。
- **归一化：** 将 $\alpha$ 向量代入 Dirichlet 分布生成器。

### 模块 C：自适应 ABC 采样 (Adaptive ABC Algorithm)

**目标：** 解决“参数拍脑门”和“样本不足”的问题。

**算法伪代码 (Python/Matlab 逻辑):**

```
For each Season S, each Week W:
    1. 确定规则 Rule_Set (Rank或Percent, 是否有Save).
    2. 设定目标样本量 Target_N = 2000.
    3. 初始化可行解集合 Posterior_Samples = [].
    4. 初始化总尝试次数 Trials = 0.
    
    While len(Posterior_Samples) < Target_N:
        # a. 从先验采样 (Dirichlet)
        # alpha 由模块 B 计算得出
        V_candidate = Dirichlet(alpha) 
        
        # b. 模拟比赛结果
        Eliminated_Simulated = Apply_Rule(Judge_Scores, V_candidate, Rule_Set)
        
        # c. 拒绝采样 (Rejection Step)
        Match = False
        If (S < 28):
            # 直接比对淘汰者
            If Eliminated_Simulated == Real_Eliminated: Match = True
        Else (S >= 28):
            # Bottom 2 逻辑
            # 只要真实淘汰者出现在模拟的 Bottom 2 中，即视为可行
            # (因为评委的主观救人是一个随机变量，我们只求"理论上有可能")
            If Real_Eliminated in Simulated_Bottom_2: Match = True
            
        If Match:
            Posterior_Samples.append(V_candidate)
            
        Trials += 1
        
        # 安全中止：如果跑了 100万次还没凑够，说明规则或先验严重偏离
        If Trials > 1,000,000: Break 
    
    5. 输出：
       - Posterior_Samples (这2000个分布就是你的后验)
       - Acceptance_Rate = Target_N / Trials (用于论文里衡量模型置信度)
```

### 模块 D：不确定性与结果量化 (Post-Processing)

**目标：** 回答题目关于 "Certainty" 的问题。

[cite_start]不要只给一个平均值！题目明确问了 "is that certainty always the same?" 。

1. **点估计 (Point Estimate):** 计算 2000 个样本的均值 $\hat{V} = \frac{1}{N}\sum V_k$。
2. **不确定性区间 (HDI):** 计算 **95% 最高密度区间 (Highest Density Interval)** 或简单的 **2.5% - 97.5% 分位数**。
3. **确定性指标 (Certainty Metric):**
   - 定义 $C_t = \frac{1}{\text{Standard Deviation of } V_{fan}}$.
   - **图表：** 画出 $C_t$ 随 Week 的变化曲线。
   - **预期结论：** 随着比赛进行（人数 $N$ 减少），解空间变小，确定性通常会上升；但在某些评委打分极其接近的周，确定性会突然下降（因为任何微小的票数波动都会改变排名）。
