# 2. 特征工程（“预处理能力”最能拉开差距的地方）

## 2.1 明星特征（固定效应候选）

来自题面数据字典：年龄、行业、地域等

## 2.2 周内动态特征（让模型更像“综艺规律”）

- `Performance_trend`：本周 `judge_total` 相对上周变化（黑马效应）

## 2.3 标准化口径（强烈建议）

为避免不同赛季/主题周“分数通胀”，建议对评委分做 **周内标准化**：

- `judge_z = (judge_total - mean_week) / sd_week`
  这样能把“某周整体打分更高”从模型里消掉。

# 3. 核心模型（主线）：混合效应模型做“归因剥离”

你们的目标不是预测，而是解释：**Pro 带飞 vs 明星自带属性**。

混合效应（Mixed Effects）是最自然的工具：同一职业舞者会带很多明星；同一明星跨多周重复观测——数据天然“分组相关”。（可以用 lme4 的经典论文作方法引用）([统计软件杂志](https://www.jstatsoft.org/article/view/v067i01?utm_source=chatgpt.com))

## 3.1 模型 A：评委视角（技术分）

**因变量**：`judge_z`（推荐）
**模型**（示意）：

[
judge_z_{i,t} =
\beta^\top X_i + \beta_w f(week_t) + u_{\text{season}} + u_{\text{pro}} + u_{\text{celebrity}} + \epsilon
]

- (X_i)：Age、Industry、Region 等
- (u_{\text{pro}})：你要的 **Pro Buff**（随机截距）
- (u_{\text{season}})：赛季随机截距（控制赛季差异）
- (u_{\text{celebrity}})：控制同一明星跨周相关性

> 输出 1：固定效应系数（解释“哪些明星特征影响评委分”）
> 输出 2：每个 pro 的随机效应 + 置信区间（“谁最会带人”）

## 3.2 模型 B：观众视角（人气票）

粉丝票是比例 (0,1)，两条路线任选一条：

### 路线 B1（更易实现）：logit 变换 + 混合效应

- 定义：`fan_logit = log((fan_share+ε)/(1-fan_share+ε))`（ε=1e-6 防 0/1）
- 模型：

[
fan_logit_{i,t} =
\gamma^\top X_i + \delta \cdot judge_z_{i,t} + \gamma_w f(week_t) + v_{\text{season}} + v_{\text{pro}} + v_{\text{celebrity}} + \eta
]

这里加上 `judge_z` 很关键：能回答题面那句“粉丝是否也受舞蹈质量影响”。

### 路线 B2（更“统计范儿”）：Beta Regression（可写成量（更规范）。([ime.usp.br](https://www.ime.usp.br/~sferrari/beta.pdf?utm_source=chatgpt.com))

如果你们实现困难：主文用 B1，附录/扩展提一句 B2 即可（不强求全做）。

# 4. 关键问法：这些因素对 Judges vs Fans “是否相同”怎么严谨回答？

你们要做的不只是“看起来不一样”，而是给出一个清晰判据：

## 4.1 主图：哑铃图（Dumbbell Plot）

- 同一特征画两点：Judges 系数 vs Fans 系数（都用标准化系数）
- 线越长，分歧越大；一正一负是“强冲突特征”

## 4.2 更硬核：交互项检验（建议写进论文）

把数据“堆叠”成两份 outcome（judge 与 fan），加 `OutcomeType` 指示变量，并对每个特征加交互项：

- `Age * OutcomeType`, `Industry * OutcomeType` …
  交互项显著 ⇒ “影响方式不同”（一句话就能封神）。

# 5. “走多远”模型（第二主线）：生存分析 Cox

题面问 “impact how well a celebrity will do in the competition”，最直观就是：**被淘汰的风险**。

## 5.1 数据构造

- `duration = 存活周数`（从 Week1 到 淘汰周）
- `event = 1` 表示淘汰；冠军/进决赛未淘汰为 `eve(t|Z) = h_0(t)\exp(\theta^\top Z)
  ]
- (Z)：Age、Industry、Region、（可选）Pro Buff（来自混合效应估计）、平均 judge_z、平均 fan_share
- 输出：hazard ratio（>1 加速淘汰；<1 延缓淘汰）

Cox 模型是经典生存分析方法（可引用原始 Cox 1972）。([JSTOR](https://www.jstor.org/stable/2985181?utm_source=chatgpt.com))
