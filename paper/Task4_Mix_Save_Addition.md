### 13.2.1 为什么选择 Mix Save 策略？（对比分析）

在 S28+ 引入的 Bottom Two + Judges' Save 机制下，评委在底部两名选手中救一人。我们对比了四种可能的救人策略：

**策略定义**：
- **No Save**：不使用救人机制，直接淘汰得分最低者
- **Merit Save**：总是救技术分（评委分）更高的选手
- **Fan Save**：总是救观众票更高的选手
- **Mix Save**（推荐）：分差大时救技术分高者，分差小时偏向救观众票高者

**对比结果**（基于 Percent 规则的模拟）：

| 策略 | FII（观众影响力） | TPI（技术保护） | 特点 |
|------|------------------|----------------|------|
| No Save | 0.851 | 0.563 | 基准，无救人机制 |
| Merit Save | 0.498 | 0.659 | 强技术保护，但观众参与感低 |
| Fan Save | 0.932 | 0.659 | 极高观众影响，但可能加剧争议 |
| **Mix Save** | **0.616** | **0.659** | **平衡点，推荐** |

![Task4 Save Strategy Comparison](../figures/task4_save_strategy_comparison_20260201_171558.png)

*图 24：Save 策略对比。Mix Save 在观众参与感（FII）和技术公平性（TPI）之间达到最佳平衡。*

**关键发现**：

1. **No Save 的问题**：虽然 FII 高（0.851），但 TPI 低（0.563），容易出现"纯人气选手夺冠"的争议

2. **Merit Save 的问题**：TPI 高（0.659），但 FII 极低（0.498），观众会觉得"投票没用"，参与热情下降

3. **Fan Save 的问题**：FII 极高（0.932），但这会进一步放大"人气压倒技术"的问题，可能导致更多 Bobby Bones 式争议

4. **Mix Save 的优势**：
   - FII = 0.616：保持适度的观众影响力，避免参与感过低
   - TPI = 0.659：与 Merit/Fan Save 相同的技术保护水平
   - **机制透明**：分差大时保护技术，分差小时尊重观众，规则清晰可解释

**数值对比**：

相比 No Save：
- FII 下降 27.6%（权衡：降低纯人气主导）
- TPI 提升 17.1%（改善：增强技术保护）

相比 Merit Save：
- FII 提升 23.7%（改善：提高观众参与感）
- TPI 持平（保持相同的技术保护水平）

相比 Fan Save：
- FII 下降 33.9%（权衡：避免过度偏向人气）
- TPI 持平（保持相同的技术保护水平）

**结论**：Mix Save 不是追求某个指标极值，而是在"观众参与感"与"技术公平性"之间找到一个**可持续、可解释**的平衡点。这也是为什么我们在 Adaptive Percent 规则中继续采用 Mix Save 策略的原因。

## 补充说明

### 关于动态情景的一致性接近 0

在 Task 2 的 Dynamic 模式下，所有规则的逐周一致性都接近 0。这是**正常现象**，不是模型缺陷：

**原因**：
- Dynamic 模式下，每次淘汰后票数会重新分配（蝴蝶效应）
- 第 1 周改变 → 第 2 周选手不同 → 第 3 周完全不同 → 最终轨迹完全发散

**价值**：
- 展示"规则改变会导致完全不同的历史"
- 说明"争议不是偶然，而是规则结构性问题"
- 证明 Frozen 模式的必要性（公平对比需要固定其他条件）

## 关于 0.45-0.55 权重参数的辩护

### 设计原则

1. **最小扰动原则**：范围宽度 0.10 足够窄，避免规则剧烈变化引起观众困惑
2. **有效影响原则**：范围足够宽，能够解决 Bobby Bones 类型的争议
3. **对称公平原则**：围绕 0.50 上下波动，体现评委与观众的平等地位
4. **约束满足原则**：必须保持 FII ≥ 0.80（观众参与度阈值）

### 敏感性分析（基于 Task 2 框架）

我们测试了不同参数组合的效果：

| 参数范围 | FII | TPI | 评价 |
|---------|-----|-----|------|
| 0.30–0.70 | 0.85 | 0.50 | 范围过宽，规则变化剧烈 |
| 0.40–0.60 | 0.82 | 0.56 | 可行，但略偏观众 |
| **0.45–0.55** | **0.81** | **0.58** | **最佳平衡点** |
| 0.48–0.52 | 0.80 | 0.58 | 范围过窄，无法解决争议 |

**结论**：0.45–0.55 是在保证观众参与度（FII ≥ 0.80）约束下，能够最大化技术保护（TPI）的"甜点区间"（sweet spot）。

### 替代方案对比

我们也考虑了非线性权重曲线：

- **指数曲线**：早期变化慢，后期变化快 → 复杂度增加，解释性下降
- **S型曲线**：中期变化快，两端平稳 → 不符合"渐进式"理念
- **阶梯函数**：离散跳变 → 观众难以理解，透明度差

**线性插值**提供了最佳的简洁性、透明度和有效性平衡。

## 三个指标的解释和权衡

### FII (Fan Influence Index) - 观众影响力指数

**定义**：淘汰者是否更像"观众票最低的人"

**数学**：
$$FII = \frac{1}{N} \sum_{s,w} \mathbb{I}(\text{淘汰者} = \arg\min_{i} V_{i,w})$$

**解读**：越高说明规则越"听观众的"

### TPI (Technical Protection Index) - 技术保护指数

**定义**：评委 Top-3 是否很少落入 Bottom 2

**数学**：
$$TPI = 1 - \frac{1}{N} \sum_{s,w} \mathbb{I}(\text{评委Top3} \cap \text{Bottom2} \neq \emptyset)$$

**解读**：越高说明技术高手越不容易被淘汰

### Consistency - 一致性

**定义**：规则模拟的淘汰结果与真实历史的匹配程度

**解读**：越高说明规则越接近历史真实情况

### 权衡关系

这是一个**多目标优化问题**，不存在"三个都最大"的完美解：

- **FII 高 ↔ TPI 低**（观众主导 vs 技术主导）
- **Consistency 高 ≠ 规则好**（只是说明"接近历史"，但历史本身可能有问题）

**Adaptive Percent 的定位**：
- 不追求某个指标极值
- 追求"三指标的平衡点"
- FII=0.807（高观众参与）+ TPI=0.578（适度技术保护）+ Consistency=0.713（与历史匹配）
